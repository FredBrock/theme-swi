颜色角色
颜色角色被分配给特定的用户界面元素,页面里的按钮、标题、背景、卡片、输入框这些 UI 元素，每一类都固定绑定一个颜色角色。

它们具有语义名称，例如 “主色”、“主色上的颜色” 和 “主容器”
主色：品牌主色调（按钮、重点强调色）
主色上的颜色：在主色背景上显示的文字 / 图标色（保证可读）
主容器：页面大背景、卡片底色


## 动态色彩
动态色彩，就是系统 / APP 自动从手机壁纸或者应用里的图片内容里提取主色调，自动生成一套适配、好看、还符合阅读舒适度的配色，直接套用在整个 UI 界面上。
1. 从用户的壁纸或应用内内容中提取一种颜色，并创建一个易于访问的配色方案
2. 分配给用户界面中的元素
3. 如果用户的壁纸或应用内内容发生更改，用户界面中的颜色也会随之更改

# 静态颜色

静态颜色就是固定不变的 UI 配色，不会跟着手机壁纸、应用里的图片自动变色。
逐句拆解
不会根据用户壁纸或应用内内容而改变的 UI 颜色和动态色彩相反，不受壁纸、页面图片影响，始终保持原本的颜色。


HCT 的颜色空间
它使用三个维度定义所有颜色：色调、色度和明度。

1. 色相（Hue，H）
就是是什么颜色。比如：红、橙、黄、绿、蓝、紫，只管色彩种类，不管深浅、艳淡。
色相用0到360之间的数字来量化.

2. 饱和度（Chroma，C）
就是颜色艳不艳。
数值高：颜色鲜艳、浓郁
数值低：发灰、暗淡、接近黑白
色调用一个介于 0（纯黑，无亮度）到 100（纯白，完全亮度）之间的数字来量化。


3. 色调（Tone，T）
专指明暗深浅，范围 0～100：
T=0：纯黑
T=100：纯白
中间数值：从深到浅的过渡
只控制亮度，不改变色相、不改变鲜艳度。
一句话总结
HCT 就是用三个维度把颜色拆死：H 决定是什么色 → C 决定艳不艳 → T 决定深还是浅
和传统 HSV 区别
普通 HSV 亮度不考虑人眼视觉感知；HCT 是适配人眼观感的，做 UI 更友好，自动保证文字对比度、适配深浅主题，是 Material 3 动态色彩、静态颜色的底层标准。
直白举例
固定色相 = 蓝色，拉高饱和度 = 深蓝色更鲜艳，调低色调 = 变浅蓝；全程还是蓝色，只是艳度、明暗变了。
色调用一个介于 0（纯黑，无亮度）到 100（纯白，完全亮度）之间的数字来量化。




1. Surface
释义：表面层 / 背景底色用途：用作页面背景、大面积低视觉权重的区域底色。
2. Primary / Secondary / Tertiary
释义：主色 / 次色 / 第三辅色用途：强调或弱化前景元素的点缀强调色；Primary 最强强调，Secondary 中等，Tertiary 最弱。
3. Container
释义：容器色用途：给按钮、卡片等前景元素做填充底色；禁止单独用来做文字、图标颜色。
4. On
释义：附着色（上层文字 / 图标色）规则：以 On 开头的角色，代表叠在对应底色上方的文字、图标颜色。例：on primary = 放在主色底色上的文字 / 图标色。
5. Variant
释义：变体色 / 弱化版色规则：带 Variant 后缀的角色，是原颜色的低强调、弱化替代版本。例：outline variant 是普通轮廓色的更浅、更低调版本
